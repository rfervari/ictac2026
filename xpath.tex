\section{Functional XPath with Data: $\xpath$} 
\label{sec:dxpath-f}

We begin presenting a fragment of the XPath language considered in~\cite{ICDT14Jair,KR16,AbriolaDFF17,AbriolaBFF18}, which we call $\xpath$. The fragment is inspired by the modal logic $\knaltone$ which has $n$ modal operators interpreted as partial functions~\cite{blackburn2001modal}. 
In brief, $\xpath$ is a two-sorted language which consists of  \emph{path} expressions and \emph{node} expressions.
Regarding path expressions, we restrict our attention to \emph{single steps} and \emph{composition}, which are interpreted as partial functions.  
We do not impose restrictions on node expressions.
These design decisions are aimed at establishing connections with some DL with concrete domains. 

In what follows, we assume that $\props = \set{p_1, \dots, p_n, p, q, r, \dots}$ is a countable set of \emph{propositional symbols}, $\mods=\set{\apath_1, \dots, \apath_n}$ is a finite set of \emph{modal labels}, and $\cmps = \set{\cmp_1, \dots, \cmp_m}$ is a finite set of \emph{comparison labels}. 

\begin{definition} [Language]\label{def:xpath:lang}
    The sets $\pexp = \{\alpha, \beta, \dots \}$ of path expressions, and $\nexp = \{\varphi, \psi, \dots \}$ of node expressions are defined by the grammar:
    \begin{align*}
        \alpha, \beta & \Coloneqq
            \apath \mid
            \alpha \beta  \\
        \varphi, \psi & \Coloneqq
            p \mid
            \lnot \varphi \mid
            \varphi \land \psi \mid
            \tup{\alpha =_\cmp   \beta} \mid
            \tup{\alpha \neq_\cmp   \beta},
      \end{align*}
     In this grammar, $p \in \props$, $\apath \in \mods$, $\cmp \in \cmps$. Other Boolean connectives are defined as usual. We define $[\alpha=_\cmp\beta]$ as $\neg\tup{\alpha\neq_\cmp\beta}$, and $[\alpha\neq_\cmp\beta]$ as $\neg\tup{\alpha=_\cmp\beta}$. 
\end{definition}

The semantics of $\xpath$ is given in terms of \emph{data models}.

\begin{definition}[Data Model] \label{def:model:xpath}
     A \emph{data model} is a structure defined as
     $\model = \tup{ \N, \D, \set{\R_{\apath}}_{\apath \in \mods}, \valuation,\set{\data_\cmp}_{\cmp \in \cmps} }$, where:
    \begin{enumerate}
        \item 
            $\N$ is a non-empty set of \emph{nodes};
        \item 
            $\D$ is a non-empty set of \emph{data values};
        \item 
            for each $\apath \in \mods$, $\R_{\apath}$ is a \emph{functional} relation on $\N$;
        \item 
            $\valuation: \props \to 2^\N$ is the valuation function;
        \item 
            $\data_\cmp:  \N \to \D$ is a function which assigns a data value to each node of the model with respect to $\cmp\in\cmps$.
    \end{enumerate}
    We use $\R_\apath(n) = m$ to indicate $(n,m)\in\R_\apath$, and say that $\R_\apath(n)$ is \emph{defined}. If such an $m$ does not exist, we say  $\R_\apath(n)$ is \emph{undefined}.
\end{definition}

Data models are a particular case of the data graphs from, e.g.,~\cite{ICDT14Jair}, in which the accessibility relations are partial functions. To properly establish the connection between XPath and DLs with concrete domains, we assume that $\D$ is a not empty common to all data models.

%As discussed, this fragment is enough to describe some interesting situations. Moreover, we will show that it can be used as a description language enjoying nice computational properties.

\begin{definition} \label{def:xpath:semantics}
    Let $\model=\tup{\N, \D, \set{\R_{\apath}}_{\apath \in \mods}, \valuation,\set{\data_\cmp}_{\cmp \in \cmps}}$ be a data model. 
    We define the semantics of path and node expressions as:
    \[\begin{array}{l@{\, } c l}
        \truthset{ \apath }{\model}  
            & = & \R_\apath \\
        \truthset{ \alpha\beta }{\model}  
            & = & \{(m,n)\mid \text{exist } l \text{ s.t. } (m,l)\in\truthset{\alpha}{\model} \text{ and } (l,n)\in \truthset{\beta}{\model}\} \\
        \truthset{ p }{\model}  & = & \valuation(p)  \\
        \truthset{ \lnot \varphi }{\model}  
            & = & \N \setminus \truthset{ \varphi }{\model} \\
        \truthset{ \varphi \land \psi }{\model}  
            & = & \truthset{ \varphi  }{\model}  \cap \truthset{  \psi }{\model}
            \\    
        \truthset{ \tup{ \alpha =_\cmp \beta } }{\model}   & = & \{ m \mid  \text{exist } n,l \text{ s.t } \\
            & & \qquad (m,n) \in  \truthset{ \alpha }{\model} \text{, } (m,l) \in  \truthset{ \beta }{\model}  \text{, }  \data_\cmp(n)=\data_\cmp(l) \} \\
        \truthset{ \tup{ \alpha \neq_\cmp \beta } }{\model}   & = & \{ m \mid  \text{exist } n,l \text{ s.t } \\
            & & \qquad (m,n) \in  \truthset{ \alpha }{\model} \text{, } (m,l) \in  \truthset{ \beta }{\model}  \text{, } \data_\cmp(n)\neq\data_\cmp(l) \}.
    \end{array}\]
    Thus, the interpretation $\truthset{\alpha}{\model}$ of a path expression is a partial function on $\N$, and the interpretation $\truthset{\varphi}{\model}$ of a node expression is a subset of $\N$. 
    A node expression $\varphi$ is \emph{satisfiable} iff there is a data model $\model$ such that $\truthset{\varphi}{\model} \neq \emptyset$. Finally, we write $\model,n\vDash\varphi$ (resp. $\model,n,m\vDash\alpha$) iff $n\in\truthset{\varphi}{\model}$ (resp. $(n,m)\in\truthset{\alpha}{\model}$).
\end{definition}
